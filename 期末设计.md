#  超市进销存管理系统 — 期末设计

> **技术栈：** ASP.NET Web Forms + .NET Framework 4.8.1 + PostgreSQL + Entity Framework 6  
> **架构原则：** 数据库纯存储，Service 层承载全部业务逻辑

---

## 一、项目概述

### 1.1 功能模块
```
超市进销存管理系统
├── 前台页面 (无需登录)
│   ├── 超市品牌介绍 + 商品浏览 + 购物车结算
│   └── 匿名快递填写
```

### 操作 1：用户登录与鉴权

**用户操作：**
1. 访问系统任意页面 → 未登录则自动跳转登录页
2. 输入用户名和密码
3. 点击"登录"按钮

**Service 层处理（UserService.Login）：**
1. 从数据库查询 username 匹配的用户，不存在则返回 null
2. 解析存储的密码哈希：格式为 `{16字节盐}.{64字节SHA256哈希}`
3. 用存储的盐重新计算 SHA256(盐 + 输入密码)
4. 比对计算结果与存储的哈希值
5. 匹配成功 → 更新 lastLoginAt = 当前时间 → 返回用户对象
6. 匹配失败 → 返回 null

**AuthHelper.SetLogin 处理：**
1. 写入 Session：`Session["UserId"]` / `Session["Role"]` / `Session["DisplayName"]`
2. 生成 Guid Token（32 位 hex 字符串）
3. Token 存入 Application：`Application["Token_{guid}"]`，设置 30 分钟过期
4. Token 写入 Cookie：`Response.Cookies["AuthToken"]`，HttpOnly，30 分钟过期
5. 按角色跳转：操作员 → Cart 购物车，管理员/店长 → Products 商品管理

**AuthHelper.IsLogin 处理（每次请求）：**
1. 检查 `Session["UserId"]` 是否存在 → 若存在则返回 true
2. 若不存在 → 调用 TryTokenLogin()
3. TryTokenLogin：读 Cookie["AuthToken"] → 查 Application 验证有效性
4. 有效 → 恢复 Session 并返回 true；无效/过期 → 返回 false → 跳转登录页

```
登录流程：
 ┌──────────┐    无Session    ┌──────────────┐    无Cookie     ┌──────────┐
 │检查Session │ ──────────► │ 检查Cookie     │ ──────────► │ 显示登录页  │
 │["UserId"]  │              │["AuthToken"]   │              │           │
 └────┬───────┘              └───────┬────────┘              └────┬─────┘
      │存在                         │有效                          │
      ▼                             ▼                       输入用户名密码
   已登录                      恢复Session                       │
      │                             │                            ▼
      └──────────┬──────────────────┘                UserService.Login
                 │                                   SHA256(盐+密码)比对
                 ▼                                        │
          SetLogin()                              成功 → SetLogin
          Session+Token+Cookie                    失败 → 提示错误
                 │
                 ▼
          按角色跳转
          操作员→Cart  管理员/店长→Products
```

### 操作 2：商品管理 + 图片上传

**用户操作（新增商品）：**
1. 在商品列表页点击"新增商品" → 跳转 ProductEdit.aspx
2. 输入编号、名称、选择分类和供应商、输入单位/进价/售价/库存/最低库存
3. （可选）点击"选择文件"上传商品图片
4. 点击"保存"

**用户操作（编辑商品）：**
1. 在商品列表行点击"编辑" → 跳转 ProductEdit.aspx?id=xx
2. 修改字段 → （可选）上传新图片或勾选"删除图片"
3. 点击"保存"

**用户操作（删除商品）：**
1. 在商品列表行点击"删除" → 软删除（isActive = false）

**Service 层处理（ProductService.Add）：**
1. product.createdAt = DateTime.Now
2. product.isActive = true
3. db.Products.Add(product)
4. db.SaveChanges()

**Service 层处理（ProductService.Update）：**
1. 根据 product.id 查询现有商品 exist
2. 逐字段更新：code, name, categoryId, supplierId, unit, purchasePrice, sellingPrice, stockQuantity, minStock, imagePath, description
3. exist.updatedAt = DateTime.Now
4. db.SaveChanges()

**Service 层处理（ProductService.Delete）：**
1. 根据 id 查询商品 exist
2. exist.isActive = false（软删除，不物理删除）
3. exist.updatedAt = DateTime.Now
4. db.SaveChanges()

**图片上传处理（HandleImageUpload — 页面层）：**
1. 检查 chkDelImage：若勾选 → 删除旧图片文件 → 返回 ""
2. 检查 fuImage.HasFile：若无文件 → 返回旧路径
3. 校验扩展名：仅允许 .jpg / .jpeg / .png / .gif / .svg
4. 创建目录 `~/Content/images/products/`（若不存在）
5. 生成文件名：`prod_{DateTime.Now.Ticks}.ext`
6. fuImage.SaveAs(路径)
7. 删除旧图片文件（若存在）
8. 返回路径：`~/Content/images/products/prod_xxx.ext`

### 操作 3：购物车结算

**用户操作：**
1. 浏览商品卡片网格 → 点击任意卡片将商品加入购物车（默认数量 1，再次点击累加）
2. 在购物车列表中用 +/- 按钮调整数量，或点击"删除"移除
3. 选择客户（可选）、支付方式（现金/微信/支付宝）、输入优惠金额（可选）
4. 点击"确认结算"

**Service 层处理（SalesService.CreateSale）：**
1. 生成单号：格式 `"XS" + yyyyMMdd + 3位流水号`
   - 查询当天该类型最大单号 → 截取后3位转数字+1 → 左侧补零
   - 若当天无单 → 从 001 开始
2. 设置：order.createdBy = userId，order.status = "已完成"，order.createdAt = DateTime.Now
3. 遍历明细行 items：
   a. 根据 productId 查询 Product
   b. **校验库存**：若 product.stockQuantity < item.quantity → 返回 -1（库存不足）
   c. product.stockQuantity -= item.quantity  ← **库存扣减**
   d. product.updatedAt = DateTime.Now
   e. item.unitPrice = product.sellingPrice
   f. item.subTotal = unitPrice × item.quantity
   g. 累加到 order.totalAmount
4. order.actualAmount = order.totalAmount - order.discountAmount
5. 若选择了客户（customerId 不为空）：
   a. customer.totalSpent += order.actualAmount  ← **更新累计消费**
   b. customer.updatedAt = DateTime.Now
   c. **自动调整会员等级**：
      - totalSpent ≥ 10000 → "金卡"
      - totalSpent ≥ 5000 → "银卡"
      - 其他 → "普通"
6. order.items = items
7. db.SalesOrders.Add(order)
8. db.SaveChanges()
9. 返回 order.id（订单 ID）

**结算后快递选择（页面层）：**
- Cart 结算后显示："是否需要快递运输？" [创建快递单] [暂不需要]
- 创建快递单 → 跳转至 Shipments 快递页面

### 操作 4：进货管理

**业务流程 1：新建进货单**

**用户操作：**
1. 在进货单列表页点击"新建进货单"
2. 选择供应商、填写备注（可选）
3. 动态添加商品行（JS 实现，每行选择商品 + 填数量，可加减行）
4. 点击"提交进货单"

**Service 层处理（PurchaseService.CreateOrder）：**
1. 生成单号：格式 `"JH" + yyyyMMdd + 3位流水号`
   - 查询当天该类型最大单号 → 截取后3位转数字+1 → 左侧补零
   - 若当天无单 → 从 001 开始
2. 设置：order.createdBy = userId，order.createdAt = DateTime.Now
3. 遍历明细行 items：
   a. 根据 productId 查询 Product
   b. item.unitPrice = product.purchasePrice（取进货价）
   c. item.subTotal = unitPrice × item.quantity
   d. 累加到 order.totalAmount
   e. item.createdAt = DateTime.Now
4. order.status = "待入库"（**此时库存不变**）
5. order.items = items
6. db.PurchaseOrders.Add(order)
7. db.SaveChanges()

**业务流程 2：确认入库**

**用户操作：** 在进货单列表，对状态为"待入库"的单据点击"入库"按钮，确认弹窗后执行

**Service 层处理（PurchaseService.ConfirmStockIn）：**
1. 校验：单据状态必须为"待入库"
2. 遍历该单据的所有明细：
   a. 找到对应 Product
   b. StockQuantity += 本次进货 Quantity  ← **库存增加**
   c. PurchasePrice = 本次进货 UnitPrice  ← **更新最新进货价**
   d. UpdatedAt = 当前时间
3. 主表 Status = "已入库"
4. db.SaveChanges()

**业务流程 3：取消进货单**

**用户操作：** 在进货单列表，对状态为"待入库"的单据点击"取消"按钮

**Service 层处理（PurchaseService.CancelOrder）：**
1. 校验：状态必须为"待入库"（已入库的单据不可取消）
2. Status = "已取消"
3. 不涉及库存变动

```
进货单状态流转图：
      ┌─────────┐
      │  待入库   │ (新建后初始状态, 库存不变)
      └────┬────┘
           │
     ┌─────┴─────┐
     ▼           ▼
 ┌───────┐   ┌───────┐
 │已入库  │   │已取消  │
 │库存+N  │   │库存不变 │
 └───────┘   └───────┘
```

### 操作 8：库存预警

**用户操作：**
1. 点击侧边栏"统计报表 > 库存预警"菜单
2. 查看库存 ≤ 最低库存的商品列表

**Service 层处理（ProductService.GetStockAlerts）：**
1. 查询 products 表：WHERE stockQuantity <= minStock
2. 若 includeDeleted = false：额外过滤 isActive = true
3. 包含 Category 导航属性（含分类名）
4. 按 stockQuantity 升序排列（最缺货的排前面）
5. 返回列表

**权限差异：**
- 管理员（includeDeleted = true）：可见含软删除的预警商品
- 店长（includeDeleted = false）：仅可见活跃商品的预警

### 操作 9：销售报表与 Excel 导出

**用户操作（查看报表）：**
1. 点击侧边栏"统计报表 > 销售报表"菜单
2. 页面显示 4 个统计卡片（总销售额 / 订单数 / 平均金额 / 总优惠）
3. 下方展示 ECharts 图表（月度趋势/商品排行/分类占比/客户排行）

**用户操作（导出 Excel）：**
1. 选择导出格式下拉（CSV / Excel）
2. 点击"导出"按钮 → 浏览器下载文件

**Service 层处理（报表数据 — LoadReportData）：**
1. 调用 SalesService.GetAllCompleted() → 获取全部已完成销售单
2. 计算统计卡片：SUM(actualAmount), COUNT, AVG, SUM(discountAmount)
3. 按年月分组聚合 → 月度标签 + 月度销售额数组
4. 调用 SalesService.GetAllSaleItems() → 获取全部销售明细
5. 按商品分组 SUM(quantity) → 商品排行 Top 10
6. 按分类分组 → 分类销量分布
7. 按客户分组 SUM(actualAmount) → 客户排行 Top 10
8. 序列化为 JSON，传入 ECharts 初始化代码

**Service 层处理（Excel 导出 — ExportExcel）：**
1. 调用 XlsxWriterHelper（纯 C# 生成 OpenXML ZIP 格式）
2. Sheet 1：销售明细数据表
3. Sheet 2-5：分别用 GDI+ 绘制柱状图图片，嵌入对应 Sheet
4. 构建 OOXML 结构文件（Content_Types, rels, workbook, styles）
5. 打包为 ZIP → 写入 HTTP Response（Content-Disposition: attachment）

### 操作 10：用户管理（含店长权限限制）

**用户操作（新增用户）：**

1. 调用 HashHelper.HashPassword(password) 加盐哈希密码
   - 生成 16 字节随机盐（hex 编码）
   - 计算 SHA256(盐 + 密码)（hex 编码）
   - 存储格式：`"{盐}.{哈希}"`
2. user.passwordHash = 哈希结果
3. user.isActive = true
4. user.createdAt = DateTime.Now
5. db.Users.Add(user)
6. db.SaveChanges()

**Service 层处理（UserService.Update）：**
1. 根据 user.id 查询现有用户 exist
2. 若传入新密码（非空）→ 重新加盐哈希 → 更新 passwordHash
3. 更新 username, displayName, role
4. exist.updatedAt = DateTime.Now
5. db.SaveChanges()

**Service 层处理（UserService.Delete）：**
1. 根据 id 查询用户 exist
2. exist.isActive = false（软删除）
3. exist.updatedAt = DateTime.Now
4. db.SaveChanges()

**店长权限限制（页面层 UserEdit.aspx.cs）：**
- Page_Load 中检查 IsManager()：清空 ddlRole，仅添加 "操作员"
- 编辑管理员用户时：隐藏编辑/删除按钮 + 页面跳转拦截 + 保存拦截
- GvUsers_RowDataBound：店长看到管理员行 → btnEdit/btnDel.Visible = false
- GvUsers_RowCommand("EditItem")：店长+目标管理员 → return
- BtnSave_Click：店长+旧角色=管理员 → return
- 店长保存时：allowedRole = "操作员"（强制，忽略下拉选择）

## 五、业务规则

| 规则 | 说明 |
|------|------|
| 单号生成 | JH/XS + yyyyMMdd + 3位流水，KD + yyyyMMdd + 4位流水 |
| 会员升级 | totalSpent ≥ 10000→金卡，≥ 5000→银卡，< 5000→普通 |
| 库存扣减 | 销售时：stockQuantity -= qty，不足返回 -1 |
| 库存增加 | 进货入库时：stockQuantity += qty |
| 软删除 | Product/User 设置 isActive=false，管理员仍可查看 |
| 图片上传 | 仅允许 jpg/png/gif/svg，路径 `~/Content/images/products/` |
| 密码存储 | SHA256(16字节随机盐 + 密码)，格式 {盐}.{哈希} |
| Token 持久 | 30分钟过期，Cookie + Session + Application 三存 |
