# 超市进销存管理系统 — 软件文档

> 版本：v2.0  
> 技术栈：ASP.NET Web Forms + .NET Framework 4.8.1 + PostgreSQL + Entity Framework 6  
> 架构原则：数据库纯存储，Service 层承载全部业务逻辑

---

## 一、项目概述

超市进销存管理系统是一套完整的 ERP 系统，覆盖前台超市展示、商品管理、供应商/客户管理、进货/销售、库存预警、销售报表、用户权限等核心功能。

### 功能模块

```
超市进销存管理系统
├── 前台页面
│   ├── 超市介绍 (Hero + 特色卡片)
│   ├── 商品浏览 (卡片网格 + 搜索)
│   └── 购物车结算 (无需登录)
│
├── 系统管理
│   ├── 用户登录（Token 持久登录，30 分钟过期）
│   ├── 三级角色权限（管理员 / 店长 / 操作员）
│   └── 用户管理（CRUD + 软删除）
│
├── 基础数据
│   ├── 商品管理（CRUD + 分类 + 供应商关联 + 图片上传 + 软删除）
│   ├── 分类管理（CRUD）
│   ├── 供应商管理（CRUD + 搜索）
│   └── 客户管理（CRUD + 会员等级 + 累计消费）
│
├── 业务管理
│   ├── 购物车（卡片网格选择商品 → 修改数量 → 结算生成销售单）
│   ├── 进货管理（新建进货单 + 确认入库 + 取消）
│   └── 销售管理（新建销售单 + 库存扣减 + 会员等级自动更新）
│
└── 统计报表
    ├── 库存预警（库存 ≤ 最低库存）
    └── 销售报表（ECharts：月度趋势 + 商品排行 + 分类占比 + 客户排行）
```

## 二、系统架构

```
┌─────────────────────────────────────┐
│           Views (.aspx+.cs)         │  ← 页面层：仅调用 Service
├─────────────────────────────────────┤
│         Services (Service层)        │  ← 业务逻辑：唯一访问 DAL 的层
├─────────────────────────────────────┤
│        DAL (ErpDbContext)            │  ← 数据访问：EF6 DbContext
├─────────────────────────────────────┤
│    Entities + EntitiesMaps          │  ← 实体 + Fluent API 映射
├─────────────────────────────────────┤
│         PostgreSQL                   │  ← 数据库
└─────────────────────────────────────┘
```

### 分层约束

| 层 | 允许依赖 | 禁止 |
|---|---|---|
| Views | Services, Helpers | DAL |
| Services | DAL, Entities | — |
| Helpers | Session, Application | DAL |

## 四、权限体系

| 角色 | 权限范围 | 可见数据 |
|---|---|---|
| 管理员 | 全部页面含用户管理 | 含软删除/已禁用 |
| 店长 | 全部页面（不可编辑管理员，仅可创建操作员） | 仅活跃数据 |
| 操作员 | 仅购物车 | 仅活跃数据 |

实现：
- `AuthHelper.IsLogin()` — 先查 Session，再查 Token（Cookie → Application）
- `AuthHelper.RequireRole("管理员,店长")` — 页面级权限拦截
- `AuthHelper.IsStaff()` — 操作员判断，自动跳转购物车
- `SidebarHelper.Build(active, role)` — 侧边栏按角色显示菜单

## 五、认证机制

### Token 持久登录

| 存储 | 用途 | 生命周期 |
|---|---|---|
| Cookie `AuthToken` | 跨 Session 持久化 | 30 分钟，HttpOnly |
| Session `AuthToken` | 快速恢复 | 随 Session |
| Application `Token_{guid}` | 验证有效性 | 30 分钟，到期自动清理 |

流程：
```
登录 → 生成 Guid Token → Cookie + Session + Application 三写
请求 → 查 Session["UserId"] → 无 → 读 Cookie → 查 Application 验证 → 恢复 Session
退出 → 三清 + Session.Abandon()
```

## 六、Service 层接口

### UserService

| 方法 | 说明 |
|---|---|
| `GetAll(includeDeleted)` | 获取用户列表 |
| `GetById(id)` | 获取单个用户 |
| `Add(user)` | 新增用户（加盐哈希密码） |
| `Update(user)` | 更新用户 |
| `Delete(id)` | 软删除（isActive=false） |
| `Login(username, password)` | 登录验证，更新 lastLoginAt |

### ProductService

| 方法 | 说明 |
|---|---|
| `GetAll(keyword, categoryId, page, pageSize, includeDeleted)` | 分页搜索 |
| `GetById(id)` | 获取单个（含分类/供应商） |
| `Add(product)` | 新增 |
| `Update(product)` | 更新（含图片路径） |
| `Delete(id)` | 软删除 |
| `GetStockAlerts(includeDeleted)` | 库存预警列表 |
| `GetActiveForSale()` | 可售商品（活跃+有库存） |

### SupplierService

| 方法 | 说明 |
|---|---|
| `GetAll(keyword, page, pageSize)` | 分页搜索 |
| `GetById(id)` | 获取单个 |
| `GetAllSimple()` | 获取全部（按名称排） |
| `Add / Update / Delete` | CRUD |

### CustomerService

| 方法 | 说明 |
|---|---|
| `GetAll(keyword, page, pageSize)` | 分页搜索 |
| `GetById(id)` | 获取单个 |
| `GetAllSimple()` | 获取全部 |
| `Add / Update / Delete` | CRUD |

### PurchaseService

| 方法 | 说明 |
|---|---|
| `GetAll(status, page, pageSize)` | 进货单列表（按状态筛选） |
| `GetById(id)` | 详情（含供应商+明细） |
| `CreateOrder(order, items, userId)` | 新建进货单（自动单号+取进货价） |
| `ConfirmStockIn(orderId)` | 确认入库（更新库存+进货价） |
| `CancelOrder(orderId)` | 取消单据 |

### SalesService

| 方法 | 说明 |
|---|---|
| `GetAll(page, pageSize)` | 销售单列表 |
| `GetById(id)` | 详情（含客户+明细） |

| `GetAllSaleItems()` | 获取全部销售明细（报表用） |
