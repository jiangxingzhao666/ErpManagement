# 超市进销存管理系统 — 网站设计说明书

> **版本：** v3.0  
> **日期：** 2025-06-13  
> **技术栈：** ASP.NET Web Forms + .NET Framework 4.8.1 + PostgreSQL + Entity Framework 6  

---

## 一、系统概述

### 1.1 项目背景

本系统是一套面向中小型超市的 ERP 管理系统，覆盖商品管理、进货销售、库存预警、会员管理、快递物流、统计报表、多角色权限等核心功能，同时提供面向顾客的前台选购页面。

### 1.2 功能全景

```
超市进销存管理系统
├── 🏠 前台页面（无需登录）
│   ├── 超市品牌介绍（Hero + 特色卡片）
│   ├── 商品浏览（卡片网格 + 搜索）
│   ├── 购物车（实时增删改 + 结算）
│   └── 快递填写（Public 匿名表单）
│
├── 🔐 认证系统
│   ├── 用户登录（加盐 SHA256 + Token 30分钟持久登录）
│   └── 三级角色权限（管理员 / 店长 / 操作员）
│
├── 📦 基础数据管理（管理员/店长）
│   ├── 商品管理（CRUD + 图片上传 SVG/JPG/PNG/GIF + 软删除）
│   ├── 分类管理（CRUD）
│   ├── 供应商管理（CRUD + 搜索）
│   └── 客户管理（CRUD + 会员等级 + 累计消费）
│
├── 💼 业务管理
│   ├── 购物车（后台：卡片网格 + 客户选择 + 优惠 + 多支付方式）
│   ├── 进货管理（新建进货单 + 确认入库 + 取消）
│   └── 销售管理（新建销售单 + 库存自动扣减 + 会员等级自动升级）
│
├── 🚚 快递管理
│   ├── 快递列表（全状态管理：待发货 / 运输中 / 已签收）
│   ├── 快递编辑（关联销售单 + 自动单号 + 七家快递公司选择）
│   └── 快递公共页（匿名用户填写收货信息）
│
├── 📊 统计报表
│   ├── 库存预警（库存 ≤ 最低库存）
│   └── 销售报表（ECharts + Excel 五 Sheet 导出含图表）
│
└── ⚙️ 系统管理
    └── 用户管理（CRUD + 软删除 + 店长仅可操作操作员）
```

## 五、核心业务流程

### 5.1 用户登录流程

```
用户访问 → 检查 Session["UserId"]
  ├─ 存在 → 按角色跳转（操作员→Cart，其他→Products）
  └─ 不存在 → 检查 Cookie["AuthToken"]
       ├─ 有效 → 恢复 Session → 按角色跳转
       └─ 无效 → 显示登录页
            └─ 提交登录 → UserService.Login → SHA256 盐哈希验证
                 └─ 成功 → SetLogin(Session + Token + Cookie) → 跳转
```

### 5.2 销售结算 + 快递流程

```
用户选购商品（Cart 或 Store 前台）
   └─ 点击结算
        └─ SalesService.CreateSale()
             ├─ 生成单号 XS+yyyyMMdd+3位流水
             ├─ 校验库存 → 不足则返回 -1
             ├─ 扣减库存 product.stockQuantity -= qty
             ├─ 计算总金额 totalAmount、实收 actualAmount
             ├─ 更新客户累计消费 + 自动升级会员等级
             └─ 返回订单 ID
                  └─ 显示结算成功 + 快递选择
                       ├─ [创建快递单] → Shipments/Edit.aspx（已登录）或 Public.aspx（未登录）
                       │    └─ 填写收件信息 → 生成 KD+yyyyMMdd+4位流水单号
                       └─ [暂不需要] → 继续选购
```

### 5.3 商品图片管理流程

```
商品编辑 → 上传图片
  ├─ 校验格式（.jpg/.jpeg/.png/.gif/.svg）
  ├─ 保存到 Content/images/products/prod_{timestamp}.ext
  ├─ 删除旧图片文件
  └─ 更新 imagePath = "~/Content/images/products/..."
       └─ 列表/购物车/前台渲染时 ASP.NET Image 控件自动解析 ~/ 路径
```

### 5.4 进货确认入库流程

```
新建进货单（选供应商 + 动态商品行）
   └─ 提交 → PurchaseService.CreateOrder
        ├─ 生成单号 JH+yyyyMMdd+3位流水
        ├─ 状态 = "待入库"（库存不变）
        └─ 写入主表 + 明细表
             └─ 点击"确认入库"
                  ├─ 校验状态 = "待入库"
                  ├─ 遍历明细 → product.stockQuantity += qty
                  ├─ 更新 product.purchasePrice
                  └─ 状态 = "已入库"
```

## 六、角色权限体系

### 6.1 三级角色

| 角色 | 标识 | 可见范围 | 操作限制 |
|------|------|----------|----------|
| 管理员 | `"管理员"` | 全部数据（含软删除/已禁用） | 无限制 |
| 店长 | `"店长"` | 仅活跃数据 | 不可编辑/删除管理员用户；新增用户仅可选"操作员" |
| 操作员 | `"操作员"` | 仅活跃数据 | 仅可访问购物车页面（侧边栏仅显示购物车入口） |

### 6.2 鉴权实现

```
AuthHelper（App_Code/Helpers/AuthHelper.cs）
├── IsLogin()          → Session["UserId"] 或 Token 恢复
├── IsAdmin()          → GetRole() == "管理员"
├── IsManager()        → GetRole() == "店长"
├── IsStaff()          → GetRole() == "操作员"
├── CanSeeDeleted()    → IsAdmin()
├── RequireLogin()     → 未登录 → 跳转 Login.aspx
├── RequireRole(str)   → 调用 RequireLogin() + 角色包含检查
├── SetLogin()         → 写 Session + 生成 Token(Cookie+Application)
└── Logout()           → 清 Token + Session.Abandon()
```

### 6.3 侧边栏菜单（SidebarHelper）

```
基础数据
  ├── 商品管理      (Products)
  ├── 分类管理      (Categories)
  ├── 供应商管理     (Suppliers)
  └── 客户管理      (Customers)
后台业务管理
  ├── 进货管理      (Purchases)
  ├── 销售管理      (Sales)
  └── 快递管理      (Shipments)
统计报表
  ├── 库存预警      (Stock)
  └── 销售报表      (Reports)
系统设置
  └── 用户管理      (Users)
门店业务
  └── 购物车        (Cart)         ← 所有角色可见
```
