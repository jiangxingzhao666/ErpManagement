# 超市进销存管理系统 — 软件文档

> 版本：v3.0 | 日期：2025-06-13  
> 技术栈：ASP.NET Web Forms + .NET Framework 4.8.1 + PostgreSQL + Entity Framework 6  
> 架构原则：数据库纯存储，Service 层承载全部业务逻辑

---

## 一、项目概述

超市进销存管理系统是一套完整的 ERP 系统，覆盖前台超市展示、商品管理、供应商/客户管理、进货/销售、快递物流、库存预警、销售报表、用户权限等核心功能。

### 功能模块

```
超市进销存管理系统
├── 前台页面（无需登录）
│   ├── 超市品牌介绍 (Hero + 特色卡片)
│   ├── 商品浏览 (卡片网格 + 搜索 + SVG 图片)
│   ├── 购物车结算 (Session 存储)
│   └── 匿名快递填写 (Public.aspx)
│
├── 系统管理
│   ├── 用户登录（SHA256 加盐哈希 + Token 30分钟持久登录）
│   ├── 三级角色权限（管理员 / 店长 / 操作员）
│   └── 用户管理（CRUD + 软删除 + 店长权限限制）
│
├── 基础数据
│   ├── 商品管理（CRUD + 图片上传 SVG/JPG/PNG/GIF + 软删除）
│   ├── 分类管理（CRUD）
│   ├── 供应商管理（CRUD + 搜索）
│   └── 客户管理（CRUD + 会员等级 + 累计消费）
│
├── 业务管理
│   ├── 购物车（卡片网格 + 客户/支付/优惠选择 + 结算 + 快递联动）
│   ├── 进货管理（新建 + 确认入库[库存增加] + 取消）
│   └── 销售管理（新建 + 库存扣减 + 客户消费更新 + 会员自动升级）
│
├── 快递管理
│   ├── 快递列表（待发货→运输中→已签收 状态流转）
│   └── 快递创建（关联销售单 + KD 自动单号 + 7家快递公司）
│
└── 统计报表
    ├── 库存预警（库存 ≤ 最低库存）
    └── 销售报表（ECharts 图表 + Excel 5 Sheet 导出含图表）
```

## 四、权限体系

| 角色 | 权限范围 | 可见数据 |
|---|---|---|
| 管理员 | 全部 21 页面 | 含软删除/已禁用 |
| 店长 | 全部页面（不可编辑管理员用户） | 仅活跃数据 |
| 操作员 | 仅购物车 + 快递页面 | 仅活跃数据 |
| 未登录 | Store 前台 + Login + 快递 Public | - |

实现：
- `AuthHelper.RequireLogin()` → 页面级登录拦截
- `AuthHelper.RequireRole("管理员,店长")` → 角色权限拦截
- `AuthHelper.IsStaff()` → 操作员跳转购物车
- `SidebarHelper.Build(active, role)` → 侧边栏按角色渲染菜单

## 五、认证机制

### SHA256 加盐密码

```
注册 → 生成 16 字节随机盐 → SHA256(盐 + 密码) → 存储 "{salt}.{hash}"
登录 → 解析存储值 → 重算 SHA256(盐 + 输入密码) → 比对
```

### Token 持久登录（30 分钟）

| 存储位置 | 用途 | 生命周期 |
|---|---|---|
| Cookie `AuthToken` | 跨 Session 持久化 | 30 分钟, HttpOnly |
| Session `AuthToken` | 快速恢复 | 随 Session |
| Application `Token_{guid}` | 验证有效性 | 30 分钟, 每10请求清理过期 |

## 六、Service 层接口

### ProductService
| 方法 | 说明 |
|---|---|
| `GetAll(keyword, categoryId, page, pageSize, includeDeleted)` | 分页搜索（名称/编号 + 分类筛选） |
| `GetById(id)` | 单个（含 category, supplier） |
| `Add(product)` | 新增 |
| `Update(product)` | 更新（含 imagePath） |
| `Delete(id)` | 软删除（isActive=false） |
| `GetActiveForSale()` | 可售商品（活跃 + 有库存） |
| `GetStockAlerts(includeDeleted)` | 预警列表（库存 ≤ minStock） |

### SalesService
| 方法 | 说明 |
|---|---|
| `GetAll(page, pageSize)` | 销售单列表（含 customer） |
| `GetById(id)` | 详情（含 customer, items.product） |
| `CreateSale(order, items, userId)` | **核心：** 生成 XS 单号 → 校验库存 → 扣减库存 → 计算金额 → 更新客户消费 + 自动升级会员等级 → 返回订单 ID |
| `GetAllCompleted()` | 已完成销售单（报表用） |
| `GetAllSaleItems()` | 全部销售明细（报表用） |

### ShipmentService
| 方法 | 说明 |
|---|---|
| `GetAll(page, pageSize)` | 快递列表（含 salesOrder.customer） |
| `GetById(id)` | 详情（含销售单完整信息） |
| `GetBySalesOrderId(salesOrderId)` | 按销售单查快递 |
| `Add(shipment)` | 新增（自动生成 KD 单号, 状态="待发货"） |
| `Update(shipment)` | 更新 |
| `UpdateStatus(id, status)` | 状态变更（发货/签收） |

## 八、业务规则摘要

| 规则 | 实现位置 | 说明 |
|------|----------|------|
| 单号生成 | PurchaseService / SalesService / ShipmentService | JH/XS/KD + yyyyMMdd + 3/4位流水号 |
| 库存扣减 | SalesService.CreateSale | 扣减 product.stockQuantity，不足返回 -1 |
| 库存增加 | PurchaseService.ConfirmStockIn | 入库时 stockQuantity += qty |
| 会员升级 | SalesService.CreateSale | totalSpent ≥ 10000→金卡, ≥ 5000→银卡 |
| 软删除 | ProductService / UserService | 设置 isActive=false 而非物理删除 |
| 图片上传 | ProductEdit / Default.aspx.cs | 校验格式，保存文件，路径写库 |
| 快递状态 | ShipmentService | 待发货 → 运输中(发货) → 已签收(签收) |
