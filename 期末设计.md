#  超市进销存管理系统 — 期末设计文档

> **技术栈：** ASP.NET Web Forms + .NET Framework 4.8.1 + PostgreSQL + Entity Framework 6  
> **架构原则：** 数据库纯存储，Service 层承载全部业务逻辑
> **实际实现：** 本设计文档描述的操作流程已全部在系统中实现

---

## 一、项目概述

### 1.1 功能模块一览

```
超市进销存管理系统
├── 前台页面 (无需登录)
│   ├── 超市品牌介绍 + 商品浏览 + 购物车结算
│   └── 匿名快递填写
│
├── 系统管理
│   ├── 用户登录 (SHA256加盐哈希 + Token 30分钟持久登录)
│   ├── 三级角色鉴权（管理员 / 店长 / 操作员）
│   └── 用户管理 (CRUD + 软删除 + 店长权限限制)
│
├── 基础数据
│   ├── 商品管理 (CRUD + 图片上传 SVG/JPG/PNG/GIF + 软删除)
│   ├── 商品分类管理 (CRUD)
│   ├── 供应商管理 (CRUD + 搜索)
│   └── 客户管理 (CRUD + 会员等级 + 累计消费)
│
├── 业务管理
│   ├── 购物车 (卡片网格 + 客户/支付/优惠 + 结算联动快递)
│   ├── 进货管理 (新建→确认入库→取消 状态流转)
│   └── 销售管理 (新建→库存扣减→会员升级 一体化)
│
├── 快递管理
│   ├── 快递列表 (待发货→运输中→已签收 状态流转)
│   └── 快递创建 (关联销售单 + KD自动单号 + 7家快递公司)
│
└── 统计报表
    ├── 库存预警 (库存 ≤ 最低库存)
    └── 销售报表 (ECharts图表 + Excel 5 Sheet含图表导出)
```

│  │                                                            │  │
│  │  ┌─────────────────────────────────────────────────────┐  │  │
│  │  │ 1. 生成单号: XS + yyyyMMdd + 3位流水                  │  │  │
│  │  │ 2. 遍历明细行:                                       │  │  │
│  │  │    for each item in items:                           │  │  │
│  │  │      product = db.Products.Find(item.productId)       │  │  │
│  │  │      ┌─────────────────────────────────────────┐    │  │  │

│  │  │      item.subTotal = unitPrice × qty                  │  │  │
│  │  │      totalAmount += item.subTotal                     │  │  │
│  │  │                                                       │  │  │
│  │  │ 3. actualAmount = totalAmount - discountAmount         │  │  │
│  │  │                                                       │  │  │

│  │  │      customer.totalSpent += actualAmount               │  │  │
│  │  │      ┌───────────────────────────────────────┐      │  │  │
│  │  │      │ if totalSpent ≥ 10000 → 金卡           │      │  │  │

## 四、Service 层业务设计

> **核心原则：所有库存变更、金额计算、数据校验、事务控制均在 Service 层完成。数据库只负责存取。**

### 4.1 ProductService — 商品服务

| 方法 | 输入 | 输出 | 业务说明 |
|------|------|------|----------|
| `GetAll(keyword, categoryId, page, pageSize, includeDeleted)` | 关键词、分类ID、页码、是否含删除 | 分页商品列表 | 支持名称/编码模糊搜索 + 分类筛选 |
| `GetById(id)` | 商品ID | 单个商品(含 Category/Supplier) | — |
| `Add(product)` | 商品对象 | 影响行数 | isActive=true, createdAt=Now |
| `Update(product)` | 商品对象(含imagePath) | 影响行数 | 全字段更新，含图片路径 |
| `Delete(id)` | 商品ID | 影响行数 | 软删除: isActive=false |
| `GetActiveForSale()` | — | 活跃且有库存的商品 | 购物车/前台使用 |
| `GetStockAlerts(includeDeleted)` | 是否含已删除 | 库存≤最低库存列表 | 预警页和仪表盘使用 |

### 4.2 SalesService — 销售服务（核心）

| 方法 | 输入 | 输出 | 业务说明 |
|------|------|------|----------|
| `GetAll(page, pageSize)` | 页码 | 销售单列表(含Customer) | id降序 |
| `GetById(id)` | 销售单ID | 详情(含Customer, Items.Product) | — |
| `CreateSale(order, items, userId)` | 订单+明细列表+用户ID | 订单ID 或 -1(库存不足) | 生成XS单号→校验库存→扣减→更新客户消费+会员升级 |
| `GetAllCompleted()` | — | 全部已完成销售单 | 报表统计使用 |
| `GetAllCompletedWithDetails()` | — | 已完成+含明细+客户 | Excel导出使用 |
| `GetAllSaleItems()` | — | 全部销售明细(含分类) | 报表图表使用 |

### 4.3 ShipmentService — 快递服务

| 方法 | 输入 | 输出 | 业务说明 |
|------|------|------|----------|
| `GetAll(page, pageSize)` | 页码 | 快递列表(含SalesOrder.Customer) | — |
| `GetById(id)` | 快递ID | 详情(含完整销售单信息) | — |
| `GetBySalesOrderId(orderId)` | 销售单ID | 对应快递单 或 null | 判断是否已创建快递 |
| `Add(shipment)` | 快递对象 | 影响行数 | 自动生成KD单号, status="待发货" |
| `Update(shipment)` | 快递对象 | 影响行数 | 更新可编辑字段 |
| `UpdateStatus(id, status)` | 快递ID + 新状态 | 影响行数 | 发货/签收状态变更 |

## 五、单号生成规则

```
进货单: JH + yyyyMMdd + 3位流水号
  示例: JH20250115001

销售单: XS + yyyyMMdd + 3位流水号
  示例: XS20250118001

快递单: KD + yyyyMMdd + 4位流水号
  示例: KD202501180001

实现逻辑 (在 Service 层):
  1. 查询当天该类型最大单号
  2. 若存在: 截取后缀转数字+1，左侧补零
  3. 若不存在: 从001(或0001)开始
```

## 六、会员等级自动计算

```
每次销售完成后，根据 Customer.TotalSpent 重新判定等级:

  TotalSpent ≥ 10000  →  金卡
  TotalSpent ≥ 5000   →  银卡
  其他                →  普通
```
